# Definition

Die Bedürfnisse der Nutzer verstehen und eine Problemhypothese entwickeln.

# Zusammenfassung

In dieser Phase werden die Ergebnisse aus der Empathie-Phase interpretiert und analysiert. Die Nutzerbedürfnisse und Erkenntnisse werden für den weiteren Prozess aufbereitet, z.B. in Form von Personas oder einer Emotional Journey Map. Der Modus hierbei ist konvergent, d.h. die Erkenntnisse werden destilliert. Kommt die Definieren-Phase zu kurz, besteht das Risiko, dass einfach beliebige Ideen gesammelt werden oder die Nutzerzentrierung verloren geht (Design für „mich selbst“).

# Ergebnis

Eine Problemhypothese die eine klare Aufgabenstellung schafft für die Ideensammlung in der nächsten Phase.

# Fragen, die in dieser Phase beantwortet werden

  • Was sind die tatsächlichen Bedürfnisse und Probleme der Nutzer?
  • Wo sollten wir den Hebel für eine Lösung ansetzen?
